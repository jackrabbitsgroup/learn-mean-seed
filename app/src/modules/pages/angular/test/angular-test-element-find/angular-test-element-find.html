Using jqLite selectors to access elements when testing

Prereqs:

Docs Testing Docs Testing Jasmine Angular Element
Angular uses jqLite (a subset of jQuery) to find elements. It's very similar to CSS selectors used to access an element - use a combination of element tags and attributes (i.e. class names), including nesting to find sub-elements, to get a hold of a particular element. Once you have the correct element, you can use it for many things when writing your tests, but some common easy ones are confirming the DOM / layout is setup properly.
Syntax for finding elements:
.find([one or more selectors])
I.e. .find('div.class-one div.class-two')
Some expect matchers:
.length
.text()
.toHaveClass()
.not.toHaveClass()

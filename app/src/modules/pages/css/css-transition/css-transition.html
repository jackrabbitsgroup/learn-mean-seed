CSS Transitions
{{pagesFullPath + 'css/transitions'}}
The transition property is used to smoothly adjust another CSS property from one value to another. Though this can be done with virtually any property, it only truly makes sense with properties that can have intermediate values, such as width and height (or anything else with a numerical value, for that matter), and also colors.

transition, as of 2013, is still a vendor-prefixed property. Thus, in Mean Seed, we use a mixin to create transition effects. The LESS version of this mixin (which is aptly named 'transition') looks like this:

.transition(background-color 2s ease 1s);

The first input is the property to be transitioned (you can use all to have the transition apply to everything).
The second input is the transition duration - the amount of time it takes for the property to transition from its old value to its new value.
The third input is a timing function, which has to do with the transition's speed. ease and linear should cover almost all your needs here; W3Schools can tell you more if that's not enough.
The fourth input is the delay - the time between when the transition is triggered and when it actually begins.

To create a transition effect, first apply the mixin to the element in question. Now the browser will 'watch' the specified property on that element; whenever the value of the property changes, the transition will kick in. Changing the property value is typically done by adjusting the element's classes somehow, which usually requires Javascript. It can also be done with pure CSS using the hover pseudo-class. Here are some examples:

Hover here!
.transition(all 5s ease 1s);

Click here, several times!
.transition(all 1s linear 0s);

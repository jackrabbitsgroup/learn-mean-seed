CSS: Selectors
{{pagesFullPath + 'css/css-selectors'}}
Selectors in CSS serve as the link between style rules and HTML elements. Elements identified (or 'selected') by the selector will beb given the styles associated with that selector.

There are dozens of ways to write a selector; they can become extremely complicated. Fortunately, however, they typically don't need to be, and good coders will often see to it that their selectors remain as simple as possible.

The most common selector by far is the class selector, which you can and should use for almost all of your CSS needs. It is flexible and easy to use: simply assign a class name to your element, and then write a CSS rule under that class name, like so:

HTML:
<div class = "selector-demo" > My Test Div </div>

CSS:
.selector-demo
{
    color: red;
    padding: 5px;
    border: solid 1px gray;
}

The code above produces this result:

My Test Div

The . in the CSS is the class selector; it tells the browser that what follows (in this case, selector-demo) is a class name that it should search for. With this CSS rule in effect, any element on the page with the class selector-demo will have red color text, 5 pixels of padding on all four sides, and a simple gray border.

You can also create a selector that targets multiple classes at once by simply writing two selectors at once, with no spaces between: .selector-demo.selector-example . This selector would only apply to elements that have both of those classes.

If you want the same style rule to apply to multiple selectors, you can use a comma-separated list: .selector-demo, .selector-example . This selector would apply to elements that have at least one of those two classes.

As mentioned before, the class selector is all you need in almost all practical cases, especially in Mean Seed. Nevertheless, you should also know about the id selector, #. It works just like the class selector, but it targets an element's id attribute instead the class attribute, and it uses a # in place of the class selector's . symbol.

You can also write selectors to target element types by simply writing the element itself, as in

div
{
    color: blue;
}

The CSS above would be applied to ALL div elements, though, which usually isn't something you want to do. Generally speaking, you want your CSS selectors to be as specific as possible, so as not to interfere with any other elements on your website.

There are a few other kinds of selectors, most notably the descendant selector and the various 'pseudo-selectors', like :hover or :first-child, but those are topics for another lesson.

W3Schools has a CSS Selector Reference which should be useful if you ever encounter a selector you don't recognize.

Mixins are not a part of standard CSS; they are an additional feature provided by the CSS preprocessors, LESS and SCSS. At their simplest, mixins are just groups of CSS lines that you can quickly insert anywhere in your CSS. (More complex mixins can also accept inputs and essentially serve as CSS functions, but you probably won't need to worry about those often.)

In LESS, every class is a mixin. Simply reference the name of a class inside a CSS rule, and the CSS lines listed under that class will be duplicated in the rule. See this page's .less file for examples.

This website uses LESS, not SCSS, so I can't provide a working example of SCSS mixins here, but I can describe the syntax. In SCSS, mixins are created using @mixin and used with @include as follows:

@mixin my-mixin
{
    color: black;
    border: solid 1px black;
}
.example-class
{
    @include my-mixin;
    [Additional styles here]
}

In the example above, when the SCSS is compiled into CSS (which is one of the many tasks performed by grunt in Mean Seed), the CSS rule for example-class would be

.example-class
{
    color: black;
    border: solid 1px black;
    [Additional styles here]
}

LESS mixins work exactly the same way - they are compiled into CSS by grunt, with the mixin's styles inserted where specified.

Mixins can be extremely useful for keeping your CSS in sync. They allow you to place the same CSS into multiple class rules, and you can later change that CSS for all of those classes with ease by simply editing the mixin. Mixins with inputs are also very helpful for handling vendor-prefixed properties. (Most such mixins have already been written for you in Mean Seed, by the way).

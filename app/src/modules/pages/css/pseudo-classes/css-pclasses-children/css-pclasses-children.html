CSS: The First-Child, Last Child, and Nth-Child Pseudo-Classes
{{pagesFullPath + 'css/pseudo-classes/css-pclasses-children'}}
The :first-child, :last-child, and :nth-child selectors are used to target elements based on their order inside their parent element. As you would expect, :first-child will only apply to an element if it is the first element inside its parent, :last-child will only apply to the last element, and :nth-child can be used to target any child (or group of children).

These selectors are most commonly used with lists, where you will frequently want to style to first or last element in the list slightly differently - for example, the first element in a vertically-oriented list might not need a margin-top for spacing purposes, but all the rest do. :first-child and :last-child spare you the trouble of giving the first or last elements a different or extra class. (This is particularly useful in conjunction with Angular's ng-repeat.)

:nth-child is a bit more complicated. It can be used to target a specific child with something like .example-class:nth-child(4), which would apply to elements with the class .example-class only if the element is the fourth child of its parent. This type of usage is fairly simple and straightforward.

:nth-child can do more, however. Instead of specifying a particular number like 4, you could write a rule to target a whole set of elements. For example, .example-class:nth-child(2n+1) would target all odd-numbered children, .example-class:nth-child(4n+3) would target elements 3, 7, 11, 15, and so on. Needless to say, this kind of versatility can make the :nth-child selector extremely useful in certain situations. In practice, however, you'll most likely rarely need to do more than target every other element, and even that isn't all that common, unfortunately.

Be careful of precedence when using these selectors together; if you have a :first-child and :nth-child selector that target the same element, (or any two -child selectors targeting the same element, for that matter), then the one that appears last in the CSS file will take precedence, overwriting any CSS properties defined by both of them.

You should also be aware of the :first-of-type, :last-of-type, and :nth-of-type selectors. These work exactly like the analagous -child selectors, except that that target elements based on their type as well as their position. For example, where .example-class:first-child targets elements that have the class .example-class and are also the first child of their parent, .example-class:first-of-type targets elements that are the first occurrence of the class .example-class in their parent, regardless of how many other sibling elements may have come before it.

CSS Color Codes

There are basically four ways to specify colors in CSS: color names, RGB codes, hex codes, or HSL codes. It doesn't matter if the color is for a border or a background or text; these color codes will all work for any and every CSS property that involves a color.

Color name is the simplest, most intuitive, and least flexible. You simply name a color, as in background-color: white;, and if the browser recognizes that color name, it will auto-convert it to a hex color code and use it. If the browser doesn't recognize the color name, though, or if the color it uses isn't quite the shade you wanted, then you're out of luck. For this reason, it's usually a better idea to use hex or RGB color codes instead of names.

rgb stands for 'Red-Green-Blue'. RGB codes are defined by three numbers - one for the amount of red, one for the amount of green, and one for blue. The lowest each number can be is 0; the highest is 255. This means that there are a total of 256 * 256 * 256 = 16,777,216 defineable colors that you can use. Here's an example of the syntax:

background-color:rgb(51, 255, 51);

Hex color codes are nothing more or less than shorthand for an RGB code. It uses a six-digit hexadecimal number to define the color, where the first two digits represent the 'red' value from the RGB code, the second two are the 'green value', and the last pair are the 'blue' value. Hex codes are much more commonly used than RGB, simply because they're shorter and easier to type. Here's that same green color above, in hex syntax this time:

background-color: #33FF33;
The '#' sign just tells the browser 'this is a hex code'.

Play around a bit with this HTML Color Picker to get a feel for how the numbers translate to colors.

(In case you were wondering, hexadecimal means 'base sixteen'. It uses the letters A-F to represent the numbers 10-15, so 2A is equal to (2 * 16) + (10 * 1) = 42, FF = (15 * 16) + (15 * 1) = 255, etc.)

HSL stands for 'Hue-Saturation-Lightness', and it won't be covered here because it's rarely used and more confusing than RGB. HSL codes can be converted to RGB codes, so I recommend avoiding HSL and sticking with RGB and hex codes. You can easily find more information about HSL online, if you need to.

There is one more color code you should be aware of: rgba. This looks and works just like an RGB code, except that it has a fourth number, for opacity. Opacity (which can also be thought of as transparency) is a number between 0 and 1; the lower this number is, the more see-through the color is. This is occasionally useful when you have elements overlapping each other, and you want to see the bottom element, too. Normally, you might use opacity (a separate CSS property) to do this, except that this will apply to the entire element. With rgba, you could, for example, make only the background color transparent:

background-color: #66FF33;

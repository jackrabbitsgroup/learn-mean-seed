CSS Layout: Inline-block Elements

There are three common ways to cause div elements to display side-by-side: Flexbox , Float , and display: inline-block.

Setting display: inline-block causes a div's width to shrink to the width of its content, and to display side-by-side next to surrounding elements rather than beginning on a new line. If an inline-block element cannot fit on the current line, it will wrap to the next line, just like text does. In fact, you can usually think of an inline-block element as a particularly big word of text: they display adjacent to surroudning content and will automatically wrap to the next line if they do not fit. They can even be aligned just like text by using the text-align property on the parent element.

This text-like behavior makes inline-block elements ideal for situations where you do not mind if your elements wrap to a new line. display: inline-block can still be useful when you do mind, but you must take care that you have enough width available to prevent unwanted wrapping.

This is often done by specifying %-based widths (and sometimes also %-based margins): if you have only 4 elements on a line, each with width:24% and no space between them, they can never wrap to a new line. This sort of strategy is particularly useful when you cannot or do not want to use flexbox, but it is not always ideal; in particular, it is very difficult to get a line of inline-block elements to touch both edges of their container. If you are off by just one pixel, the last element could wrap to a new line, which is usually disastrous for your layout.

Because unwanted wrapping can be such a page-killer, it is sometimes important to know exactly how much space is between your elements. Thus, you should be aware that if there is any white space in the html between two inline-block elements, the browser will place a space between them - just as if they were text! display:inline-block often goes hand-in-hand with another property, vertical-align. If your elements are not all the same height, you must decide how they sit next to each other. Usually, this means setting each to vertical-align:top, which aligns the elements along their top edge, or vertical-align:bottom, which aligns the elements along their bottom edge. There are other options, but they do not always behave as you might expect, so be careful.

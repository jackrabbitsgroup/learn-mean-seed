CSS Layout: The Position Property
{{pagesFullPath + 'css/css-layout-position'}}
The position property is used to specify where an element should be placed relative to its surroundings. There are four possible values: static, relative, absolute, and fixed.

static is the default value. Elements are positioned according their order in the html, just as you expect.

relative is essentially exactly the same as static, for most purposes. The one crucial difference is how they interact with internal absolute elements.

absolute elements are placed relative to their nearest non-static parent, regardless of any sibling elements. (This is why relative elements are useful.) This is the most common way to cause elements to overlap one another, and can even be used to make an element appear outside of the boundaries of its parent, because parents of absolute elements will not expand their width or height to accomodate their absolutely positioned children: as far as the parent element is concerned, absolute children have 0 width and 0 height.

fixed elements are not placed relative to their parents at all, but instead relative to the browser window. That means that when you scroll the window, fixed elements will stay where they are on the screen. Like absolute elements, fixed elements have no height and width as far as their parent elements are concerned.

absolute and fixed come with 5 more relevant properties to specify their position: top, bottom, left, right, and z-index.

The first four are numbers in px (pixels) or %. top specifies how far down the top edge of the element should be (relative to the top edge of the nearest non-static parent for absolute or to the window for fixed). left, right, and bottom do the same for the other 3 edges, though you'll rarely need to define all four of them at once. You can use negative values here to move the element outside of its parent.

z-index is a unitless number used to order overlapping siblings. If two elements overlap, the one with the higher z-index will be placed on top of the other, along with everything inside it. z-index only works on non-static elements.

CSS: The Descendant Selector

The descendant selector allows you to target child elements of a particular parent. Syntactically, it is very simple; just write a selector for the parent element followed by a selector for the child, with a space between, as in .parent .child. In this example, the selector targets elements with the class child, but only if they are also inside an element with the class parent.

You can chain this selector together as well, as in .ancestor .parent .child, in order to target only .children that are descendants of .parents that are descendants of .ancestors. You can do this as many times as you like, but if you find yourself chaining often or many times, you should probably reevaluate your strategy.

Side note: There is also the child selector >, as in .parent > .child. This works just like the descendant selector, except it targets only immediate children rather than any descendant of .parent.

The descendant selector can be very helpful, but it can also become annoying and difficult to keep track of if used too often. Thus, it should be used only when necessary. Do NOT use a descendant selector simply because you were too lazy to write a new class for the element you want to style! Rather, save it for a situation where you have no easier way to distinguish two elements than by their parents.

You will probably find that you almost never need to use the descendant selector, but on those rare occasions when you do, it can save you from a massive headache.

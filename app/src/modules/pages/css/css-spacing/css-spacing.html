CSS Spacing: Margin, Border, and Padding
{{pagesFullPath + 'css/css-spacing'}}
Adding spacing between elements is one of the most common tasks in CSS, and luckily for us, it's not difficult. The three relevant properties are margin, border, and padding.

You already know what a border is - it surrounds the outside of an element, usually as a thin, solid line. Margin is the space on the outside of the border, separating the border from any elements surrouding our element. Padding is the space on the inside of the border, separating the border from the interior of the element. This image says it all:



Each of these properties has four sides: top, bottom, left, and right. You can define all four at once with something like padding: 10px;, or you can specify each individually using margin-top, margin-left, etc.

Margin*, padding, and border width** are valued in px (pixels) or as a % of the available width. Be warned that %s do not work well, if at all, with regards to height; do not rely on % values when specifying heights, tops, bottoms, or any another vertical property, for that matter. They almost never behave as you would expect.

*There is another value for margins that can be very useful: setting margin-left: auto; margin-right: auto; will center the element horizontally in the available space. Try resizing your window - this div will stay centered. This does not work vertically, though, for the same reason that % values do not work well with heights - browsers are notoriously bad at that sort of thing.


**I said 'border width' because the border property is actually a combination of three properties: border-width, border-style, and border-color. That's why border values have three parts, as in border: 1px solid black. The order in which you write these parts does not matter.

Angular Directives: ng-show, ng-hide, ng-if
{{pagesFullPath + 'angular-basics/directives/ang-directives-show-if'}}
Angular's ng-show is a very simple but nevertheless very helpful directive, used to conditionally display an element. It takes a Javascript expression as its argument that should evaluate to either true or false, just like an expression you'd use in an if statement.

If the expression evaluates to true (or to a "truthy" value), the element will be displayed. If the expression evaluates to false (or to a "falsy" value), the element will be hidden.

ng-hide works exactly the same way as ng-show, except the roles of true and false are reversed: true values hide the element, false values show it. It exists purely for your convenience; everything that can be done with ng-hide can easily be done with ng-show as well.

ng-if is almost identical to ng-show, but it differs in one very important aspect: when ng-show receives a false expression, it hides the element using the CSS code display: none, causing the element to disappear as far as the user is concerned. ng-if however, does not merely hide elements; it obliterates them.

When ng-if receives a false expression, it removes the element from the page entirely. The element is not even rendered at all - you won't even see it in your web inspector, because it no longer exists. This means that if you want to re-show an element "hidden" by ng-if, then the browser must completely re-render the element and everything in it, which is inefficient. On the other hand, the browser will not bother to render elements "hidden" by ng-if, which is more efficient if you know that that particular element is unlikely to be displayed (or should never be displayed) to your current user.

Summary: Use ng-show (or ng-hide) for things that are likely to be frequently toggled between shown and hidden. Use ng-if for things that are never toggled or are toggled very infrequently.

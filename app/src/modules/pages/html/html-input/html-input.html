The input element is primarily used to build forms where users can input data. There are many types of inputs, each with their own rules, and constructing and retrieving a user's data from them can become difficult very quickly. For these reasons and others, inputs are by far the most complicated html element you'll deal with.

The good news for us is that Angular makes retrieving data from inputs much, much easier, and Mean Seed's angular forminput directive helps simplify things even more, so for now we can just focus on the basics.

The simplest type of input is the text input. Take a look at this page's .html file to see how it's done:



Note that this element is self-closing.

Text inputs are actually quite simple to work with (especially with Angular's ng-model), which is fortunate because they are by far the most common. You can give them classes and apply CSS styles, among other things.

There are also password inputs, which work just like text inputs except that the characters are hidden:



For multi-line text inputs, use the special textarea element. It's technically not an input, but it looks a lot like one:


Other types of inputs include radio buttons, checkboxes, and select inputs (which are again technically not an input element), to name a few. We're not going to worry about these in too much detail here, since Mean Seed's forminput directive handles a lot of the grunt work. If you need more information, of course, you can always ask the internet for examples of any particular type of input.

As you've probably guessed, the img tag is used to display an image (like a .jpg, .png, or .gif file) on the page. Img tags have three improtant attributes: src, height, and width.

Src, which is short for 'source', holds the path to the image. This is the property that tells the page what image file to display. All images should have an src attribute defined, because an image element with no image doesn't make sense!

Example usage:
<img src = 'path/to/my-image.jpg' />
Note that the img tag is self-closing.

The code above tells the browser to look up the file named my-image.jpg in the folder to in the folder path and display it on the page. You can use relative paths or absolute paths to get to your image, but relative paths can be tricky if you're not sure where you're starting from. Absolute paths are more reliable in the sense that there is no ambuiguity about where they point to, but they can often be much more of a hassle to create.

If you're working with our MEAN seed, though, you shouldn't have any trouble getting to your images - Angular and its $scope variables make that easy:

<img ng-src = 'appPathImg + "loading.gif"' />


If you don't know what Angular is yet, don't worry - just know that appPathImg is a variable that MEAN seed has defined for you, and it will get you to the app/src/common/img folder, so you can put your image files there and write your paths relative to that. ng-src is an Angular thing that works just like src, except that it lets us use Angular variables like appPathImg.

If you want to display an externally hosted image (meaning that the image file is not stored with the rest of your website files), then you need to use an absolute url. I found the Angular logo below at http://angularjs.org/img/AngularJS-large.png, so I can use that url to display the image on this page.
<img src = 'http://angularjs.org/img/AngularJS-large.png' />


If, however, that image is ever taken down, or if for any other reason that url stops leading to that image, then the image will not show up here anymore. The image will also fail to display if you are not connected to the internet, since your browser won't be able to reach that url.

The img tag's other special attributes, width and height, are exactly what you think - they let you specify the size that the image is displayed at, in pixels. You'll typically do this with CSS instead, but these attributes are there if you need them.
If omitted and there's no CSS active to alter the image's size, then the image will display at its normal resolution.
